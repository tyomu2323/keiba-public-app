# keiba-public-app v13

公開用の競馬分析ダッシュボードです。

## V13 追加内容

- 騎手詳細の距離成績を「芝」「ダート」に分離表示
- 芝距離別、ダート距離別、開催場別をそれぞれ確認可能
- 馬詳細に「基本 / 追い切り / 過去走」ジャンプボタンを追加
- 過去走の上がりを順位付きで表示
  - 1位：🥇金
  - 2位：🥈銀
  - 3位：🥉銅
- 詳細モーダルを閉じたら画面トップへ戻る
- 詳細を開くたびにモーダル内スクロール位置を先頭にリセット

## Render設定

Build Command:

```bash
npm install && npm rebuild better-sqlite3
```

Start Command:

```bash
node server/index.js
```

Environment Variables:

```text
ADMIN_USER=admin
ADMIN_PASSWORD=任意の管理者パスワード
JWT_SECRET=長いランダム文字列
NODE_ENV=production
```


## v14 追加内容
- 管理画面に「追い切り手動加点」を追加
  - 出走馬一覧と全追い切りを表示
  - タイム横のボタンで -3〜+5 を保存
  - 基準は0点
  - 追い切りは馬の状態・気持ち・動きを見るため、数字評価とは別に手動評価
- 最終表示を「自動加点 / 手動加点 / 総合点」に変更
- 自動加点ルールをユーザー理論に合わせて追加
  - 距離実績 +1
  - 同場所実績 +2
  - 右回り/左回り実績 +1
  - 小回りコース実績 +1
  - 大箱/外回り系コース実績 +1
  - 前走上がり3位以内 +1
  - 騎手直近1ヶ月勝率15%以上 +1
  - 重賞5着以内 +1
  - 前走距離延長/短縮で3着以内 +1
  - 同条件実績 +2
  - 自己条件レース +3
  - ハンデ戦は最重量を基準に1kg軽いごとに+0.5
- 距離延長/短縮が今回合うかどうかは手動判断扱いで、但し書き0点ルールとして表示


## V15 update
- 管理画面の追い切り一覧で枠/馬番をJRA配色に変更。
- 追い切り手動加点を馬単位ではなく調教タイムごとに保存するように変更。
- 加点保存時は画面全体を再描画せず、該当行と合計だけ更新して固まりにくくしました。
- レース詳細に負担斤量を追加。
- ハンデ戦斤量補正は「最重量 - 当該馬斤量」×0.5で自動計算します。


## V17 update
- 斤量補正をハンデ戦限定ではなく全レースに適用。
- 計算式: (レース内の最重量 - 当該馬の斤量) × 0.5。
- サンプルデータの斤量を 58, 57.5, 57, 56, 55.5, 55, 54, 52kg に分散。


## V18
- 斤量補正を明示的に 1kgごと+0.5 に固定。
- 表示理由に `差分kg × 0.5 = 加点` を出すようにしました。
- 計算式: `(レース内の最重量 - 当該馬の斤量) × 0.5`。
